How it works

The WEAR-RAG Pipeline

Five sequential stages transform a raw question and a set of documents into a high-quality, evidence-grounded answer. Each stage is designed to reduce noise and increase precision.

01 ✂️
Semantic Chunking

Documents split by cosine similarity between adjacent sentence embeddings — preserving semantic coherence.

02 🔍
Query Decomposition

Complex multi-hop questions broken into targeted sub-queries to improve retrieval coverage.

03 📦
FAISS Retrieval

Dense vector search using BAAI/bge-small-en embeddings. Top-20 candidates per sub-query retrieved.

04 ⚖️
Cross-Encoder Rerank

BAAI/bge-reranker-base scores each candidate for precise query-document relevance.

05 🧠
LLM Generation

Mistral 7B generates the final answer from aggregated, noise-filtered evidence context.

Components

Model Architecture

WEAR-RAG uses three pre-trained models orchestrated by the weighted aggregation pipeline. No model training required — the innovation is in how evidence is ranked and selected.

🔷
Embedding Engine
BAAI/bge-small-en

384-dimensional dense embeddings for semantic similarity. Uses BGE-specific query and passage instruction prefixes for improved retrieval accuracy.

🎯
Cross-Encoder Reranker
BAAI/bge-reranker-base

XLM-RoBERTa based sequence classifier that scores query-document relevance jointly. Significantly more accurate than bi-encoder similarity alone.

💬
LLM Generator
Mistral 7B via Ollama

7-billion parameter instruction-tuned model served locally via Ollama. Generates final answers from the weighted evidence context window.

📐
Document Processor
Semantic Chunker

Splits documents into semantically coherent chunks by detecting cosine similarity drops between adjacent sentence embeddings.

🗂️
Vector Store
FAISS IndexFlatIP

Facebook AI Similarity Search with inner product indexing. Supports AVX2 acceleration. Handles ingestion, retrieval, save, and load operations.

🔗
Query Decomposer
Rule-based + LLM fallback

Breaks multi-hop questions into targeted sub-queries using rule-based patterns first, with an optional LLM-powered fallback for complex queries.

Benchmark

HotpotQA Dataset

HotpotQA is a large-scale multi-hop question answering dataset requiring reasoning across two or more Wikipedia paragraphs. We use the distractor setting, where each question is paired with 10 documents — 2 gold supporting docs and 8 distractors.

This setting directly tests retrieval quality: the system must identify the 2 relevant documents among 8 noise documents before generating an answer.
